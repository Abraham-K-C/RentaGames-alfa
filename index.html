GamesRental

Plataforma de distribución de videojuegos construida sobre una arquitectura de microservicios con Spring Boot, Spring Cloud Eureka, API Gateway, React 18 y stack de observabilidad Prometheus + Grafana. Inspirada en la experiencia de Steam / Epic Games.

React 18 Spring Boot Spring Cloud Zustand Prometheus Grafana 13 Eureka H2 Database Vite 5

Visión General

GamesRental es un sistema distribuido end-to-end que simula una plataforma comercial de videojuegos. Cada dominio de negocio está encapsulado en su propio microservicio independiente con base de datos propia, y el frontend consume todos los servicios a través de un proxy que en producción apunta al API Gateway.

Patrón por microservicio

Cada uno de los 10 microservicios sigue exactamente la misma estructura interna:

src/main/java/com/gamesrentall/{servicio}/
├── controller/    ← REST Controller (@RequestMapping("/api/{servicio}"))
├── entity/        ← JPA Entity
├── repository/    ← JpaRepository
└── service/       ← Lógica de negocio

src/main/resources/
└── application.yml  ← Puerto, datasource H2, Eureka, Actuator

Todos los microservicios se registran automáticamente en Eureka al arrancar. El Gateway resuelve las rutas por nombre de servicio (no IP fija), lo que permite escalar instancias sin reconfiguración.

API Endpoints

Todos los servicios siguen el mismo patrón REST. Operaciones CRUD estándar más un endpoint de prueba.

Patrón común (ejemplo: catalogo-service)
GET    /api/catalogo        → Listar todos los juegos
GET    /api/catalogo/{id}   → Buscar por ID
POST   /api/catalogo        → Crear juego (body: JSON)
PUT    /api/catalogo/{id}   → Actualizar juego
DELETE /api/catalogo/{id}   → Eliminar juego
GET    /api/catalogo/test   → Health check del servicio

Endpoints específicos por dominio

Flujo de reembolso (regla de negocio crítica)

Cuando el admin aprueba un reembolso desde el panel de soporte, el sistema ejecuta 3 cambios simultáneos:
  1. Ticket → estado Resuelto
  2. Pago → badge cambia de completado a reembolsado y se resta del total de ingresos
  3. Juego se elimina de la Biblioteca del usuario afectado

Flujos de Negocio

Flujo de compra de un juego

1

Usuario navega la tienda

StorePage consume GET /api/catalogo y GET /api/ofertas para mostrar precios con descuentos activos y sus countdowns.

2

Selecciona método de pago

PaymentGateway muestra tres opciones: Tarjeta de Crédito/Débito (con validación de campos), Yape (QR simulado + código) y Plin (identidad visual propia).

3

Procesamiento del pago

POST /api/pagos con el monto efectivo (aplicando descuento si hay oferta activa). El servicio registra la transacción.

4

Juego pasa a la biblioteca

POST /api/biblioteca agrega la entrada. El estado Zustand se actualiza de forma optimista y el juego aparece inmediatamente en Biblioteca.

Flujo de reseña

1

Validación de propiedad

El botón de reseña solo se habilita si el juego está en la biblioteca del usuario (libraryIds.includes(gameId)). Si no lo tiene, ve el mensaje de bloqueo.

2

Publicación

El usuario selecciona estrellas (1-5) y escribe su texto. POST /api/resenas. Solo puede publicar una reseña por juego.

3

Gestión propia

El usuario puede editar o eliminar su propia reseña. No puede modificar las de otros. El admin solo puede monitorearlas, no alterarlas.

Guía de Arranque (DEV)

Ejecutar en este orden exacto. Cada paso depende del anterior.

Paso 1 — Observabilidad

.\1-iniciar-observabilidad.bat

Levanta Prometheus (recolección de métricas) y Grafana (dashboards).

Paso 2 — Infraestructura

.\2-iniciar-infraestructura.bat

Arranca Eureka Service Discovery (:8761) y el API Gateway (:8080).

Paso 3 — Microservicios

.\3-iniciar-microservicios.bat

Levanta los 10 microservicios (puertos 8081–8090). Cada uno se registra en Eureka automáticamente.

Paso 4 — Frontend

cd frontend
npm install
npm run dev

Arranca el servidor de desarrollo React en http://localhost:5173.

El frontend funciona completamente con datos mock sin necesidad de los microservicios. Para verificar la conexión real, usar el indicador "Servicios" en la barra de navegación y hacer clic en "Revisar".
